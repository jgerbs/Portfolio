Ransomware Incident Report

Executive Summary

This report documents a simulated ransomware security incident executed on a workstation environment. The simulation demonstrates how a common ransomware attack unfolds, how Sysmon logs appear inside a SIEM, and how an analyst performs triage, timeline reconstruction, and evidence collection in accordance with NIST 800-61, CompTIA Security+, and Google Cybersecurity Certificate best practices.

Attack Overview

The incident began when the user accessed a deceptive website disguised as a legitimate photo editor. The site delivered a malicious executable (UnapprovedApp.exe) which the user downloaded and ran. Upon execution, a script silently enumerated and encrypted files in the Documents and Pictures directories, then attempted to delete shadow copies to prevent recovery.

  • Initial Access: Social engineering via deceptive download website
  • Execution: User runs UnapprovedApp.exe
  • Discovery: Malware enumerates local files
  • Impact: Local file encryption
  • Recovery Inhibition: Shadow copy deletion attempt

Root Cause Analysis

The root cause of the incident was the user's execution of unverified software from an untrusted website. Lack of application allow-listing, insufficient user awareness training, and missing endpoint protection tools allowed the malware to run without interruption.

Recommendations

  • Implement Application Allow-Listing (WDAC or AppLocker)
  • Enhance web filtering to block deceptive download sites
  • Deploy EDR capable of blocking script-based post-install execution
  • Ensure backups are immutable and segmented
  • Provide cybersecurity awareness training
  • Enable Sysmon + centralized SIEM logging
