# Ransomware Attack Simulation

An interactive browser-based simulation of a ransomware infection chain, from initial download through file encryption, with a live Splunk-style SOC log viewer running alongside the attack.

## Overview

This demo walks through a complete ransomware scenario in a three-pane interface: a simulated browser showing how the infection starts, a simulated file system that reacts to the attack in real time, and a SOC log panel that populates with Sysmon-formatted events as the simulation progresses. The user makes decisions at key points — whether to visit a suspicious site, download an executable, and run it — mirroring the actual human choices that lead to real ransomware infections.

Nothing in this demo touches real files, real processes, or real system resources. All activity is confined to JavaScript objects in memory.

## How It Works

The demo is divided into three panes that update in sync:

**Left pane — Simulated Download**
A fake search engine ("Foogle") returns a deceptive result for a free photo editor. Clicking through leads to a convincing-looking download page for `UnapprovedApp.exe`. The user can navigate between the search results and the fake site, or go back, just as they would in a real browser.

**Center pane — Simulated File System**
An in-memory file system object holds three folders (`Documents`, `Pictures`, `Downloads`) with realistic placeholder files. When `UnapprovedApp.exe` is downloaded, it appears in the `Downloads` folder. Clicking it triggers an execution warning modal — a last-chance prompt before the simulation runs. Once confirmed, the ransomware sequence begins: files are enumerated, encrypted one by one (filenames gain a `.jackcrypt` extension and the simulated content is obfuscated), and the folder view updates after each file is processed.

**Right pane — Splunk-Style SOC Log Viewer**
Every simulated action generates a Sysmon-formatted event object with fields matching real Sysmon output: timestamp, host (`EmployeeWorkstation-01`), source (`Sysmon`), event ID, severity, process metadata, and file target. Events are tagged `INFO`, `WARNING`, or `CRITICAL` with color coding. Clicking any row expands it to show the full JSON event with VS Code-style syntax highlighting. The search bar accepts Splunk-style query syntax (`field=value` and raw keywords) to filter events.

After encryption completes, a ransom modal appears showing an encrypted content preview. A simulated payment button reverses the obfuscation and restores all files, with a final log entry recording the decryption.

## Educational Purpose

This simulation teaches how a ransomware attack unfolds from the user's perspective and the analyst's perspective simultaneously:

- How social engineering and deceptive download sites deliver malware without any technical exploit
- What a ransomware execution chain looks like step by step: download → execution → file enumeration → encryption → anti-recovery
- How Sysmon telemetry captures each phase of a malware execution in a SIEM
- Why shadow copy deletion is a standard ransomware recovery-inhibition technique
- How SOC analysts use log search and severity filtering to triage an active incident
- Why user behavior (running unapproved software) is often the root cause, not a technical vulnerability

- HTML5

No frameworks, no build tools, no external requests (the Foogle and download site content is generated entirely in JavaScript).

## Usage

1. Click **Proceed to Demo** on the intro modal
2. In the left pane, click **Visit Site** on the Foogle result
3. On the fake download page, click **Download for Windows** — `UnapprovedApp.exe` appears in the Downloads folder in the center pane
4. In the center pane, click on `UnapprovedApp.exe`
5. Read the execution warning modal — then click **Run Anyway** to trigger the simulation
6. Watch the file system update as each file is encrypted, and the SOC log populate with events in the right pane
7. When the ransom modal appears, click **Simulate Payment (Decrypt Files)** to restore the files and observe the final log entry
8. Use the search bar in the right pane to filter events — try `severity=critical` or `Encrypting` to narrow the log

## Ethical Use Notice

This demo operates entirely on fictional, in-memory data. The "file system" is a JavaScript object — no real files are read, written, encrypted, or deleted. The "executable" is a button that triggers a scripted sequence. The Sysmon logs are generated by the simulation, not read from any real system.

The social engineering scenario (deceptive search result, fake download site) is included because social engineering is the most common initial access vector for ransomware — showing it accurately is necessary to build awareness of what these attacks look like in practice.

## Limitations

- The "encryption" uses `btoa` (base64 encoding) with a string reversal — it is visually illustrative of what obfuscated file content looks like to a victim, but is not cryptographically representative of real ransomware, which uses AES or RSA
- The Sysmon event format matches real Sysmon field names and event IDs but is generated by the simulation, not captured from a real endpoint
- The Splunk search parser supports basic `field=value` and keyword filtering only — it is not a real SIEM query engine
- The simulated file system is reset on page refresh; there is no persistence
